# Saga 0.4.0-alpha.4 - "Scribe" Release Notes

Saga `0.4.0-alpha.4`, code-named **Scribe**, is an alpha build. Set your expectations to *alpha*.

Last updated: July 17, 2026.

## Compatibility

- Minimum SillyTavern version: `1.12.0`.
- `auto_update: false`; automatic updates are disabled for this alpha.
- Update manually from the repository when moving to a newer alpha build.
- This build follows `0.3.0-alpha.3` (**Voyages**) and adds a release-facing external authoring path for Saga Loredecks.

## Implemented Features

### Major Changes

- **Loredeck Builder toolkit:** Added the external `loredeck-builder` skill and `loredeck` CLI for creating, validating, and packaging Saga Loredecks outside the SillyTavern extension.
- **Staged authoring workflow:** The toolkit guides authors through intake, scope brief, evidence, Context/timeline planning, title batches, Lorecard batches, Pack Health, promotion, packaging, and final verification. Each stage is user-gated.
- **Evidence-grounded authoring:** Evidence records, acceptance/rejection workflow, source provenance, cross-deck citation reporting, and authoring guidance make the source boundary explicit before cards are drafted.
- **Multi-deck family support:** Projects can model a shared core plus era, faction, or other linked decks. Deck-scoped gates, incremental deck addition, shared Library paths, cross-deck tag resolution, and single-deck delta packages are supported.
- **CLI and extension Pack Health parity:** The external toolkit vendors the same Pack Health logic used by Saga so promotion and package verification enforce the same strict-clean release bar.
- **Validated package distribution:** The toolkit can promote drafts, build `.saga-loredeck.zip` packages, verify the final archive, and produce a distributable `.skill` bundle through the repository's release workflow.

### Documentation

- Added the [Loredeck Builder Toolkit](../loredecks/LOREDECK_BUILDER_TOOLKIT.md) reference, including command usage, workshop layout, validation boundaries, multi-deck conventions, and a minimal end-to-end example.
- Expanded the [Loredeck and Lorecard Creation Guide](../loredecks/LOREDECK_AND_LORECARD_CREATION_GUIDE.md) with the external authoring workflow and human-review responsibilities.
- Expanded the [Saga Loredeck Schema](../loredecks/SAGA_LOREDECK_SCHEMA.md) guidance around family roles, truth/reveal semantics, context requirements, and release-facing authoring contracts.
- Documented evidence truncation warnings: Fandom fetches remain capped at 3,000 characters to keep evidence records manageable, and a warning identifies when the fetched source was truncated so authors can review the complete source before accepting it.
- Updated the README, documentation index, scripts guide, plugin guide, and bundled skill references for the current authoring workflow.

### Verification

The Loredeck authoring path has focused coverage for:

- workshop initialization, resume state, project and deck-scoped gates, batch records, and incremental deck addition;
- evidence validation, acceptance/rejection, verbose unbacked-card reports, and cross-deck citations;
- cross-deck tag-parent resolution and CLI/browser Pack Health parity;
- promotion, package, single-deck package, and final archive verification;
- CLI conformance and review-artifact generation;
- the Loredeck Builder plugin bundle's sync, health, build, and archive contents.

Before tagging or packaging this alpha, run:

```powershell
node tools/scripts/run-alpha-gate.mjs
```

The focused Loredeck checks can be run individually with the `test-loredeck-*.mjs` scripts in `tools/scripts/`.

## Alpha Boundaries

- First run does not auto-open Saga; open it from the extension menu.
- Schemas and storage formats are still changing during alpha development.
- Legacy state from older local builds is not guaranteed to migrate. Export anything important before updating.
- Unsupported old imported state schemas are rejected instead of partially migrated.
- The external Builder validates structure, provenance references, and Pack Health, but it cannot determine whether an author's interpretation is faithful to a source. Human review and explicit stage approval remain required.
- Evidence fetched from external sources may be truncated at the configured record limit. Treat the truncation warning as a prompt to inspect the complete source before accepting the record.
- Imported Loredeck packages are data-only, but their lore can still affect prompts after users load and use them. Treat packages from unknown sources as untrusted prompt material.
- Direct provider key storage is available for alpha testing, but SillyTavern Connection Profiles or a backend proxy are preferred when possible.
