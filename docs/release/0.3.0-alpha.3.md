# Saga 0.3.0-alpha.3 - "Voyages" Release Notes

Saga `0.3.0-alpha.3`, code-named **Voyages**, is an alpha build. Set your expectations to *alpha*.

Last updated: June 19, 2026.

## Compatibility

- Minimum SillyTavern version: `1.12.0`.
- `auto_update: false`; automatic updates are disabled for this alpha.
- Update manually from the repository when moving to a newer alpha build.
- This build follows `0.2.0-alpha.2` (**The Big Mobile Update**). Export anything important before updating from older local builds.

## Implemented Features

### Major Changes

- **Bundled Lorepack expansion:** Saga now includes 34 additional bundled Loredecks since alpha 2: 13 One Piece arc packs from Romance Dawn through Post-Arabasta, and 21 Star Trek season packs covering TNG, DS9, and Voyager seasons 1-7.
- **Larger bundled reference library:** The bundled index now includes 77 Loredecks across Harry Potter, Middle-earth, My Hero Academia, Jujutsu Kaisen, One Piece, Star Trek, and Star Wars. The new One Piece packs add 477 entries; the new Star Trek packs add 1,669 entries.
- **One Piece pack coverage:** The One Piece family includes arc-scoped entries, Context gates, resolver aliases, tags, taxonomy, scoring, timeline data, and cover assets for Romance Dawn, Orange Town, Syrup Village, Baratie, Arlong Park, Loguetown, Warship Island, Reverse Mountain, Whisky Peak, Little Garden, Drum Island, Arabasta, and Post-Arabasta.
- **Star Trek pack coverage:** The Star Trek family adds season-scoped packs for The Next Generation, Deep Space Nine, and Voyager, with episode gates, crew/faction state entries, Context registries, tags, taxonomy, scoring, resolver data, and timeline data. TNG and Voyager packs include cover assets.
- **Story Maker reliability:** Story Maker generation now supports explicit cancellation, provider timeout handling, stale active-run recovery, and safer interrupted-generation cleanup so late provider responses do not overwrite the current session state.
- **Deck Maker generation hardening:** Deck Maker generation has been split into tested controllers for settings, sessions, progress, recovery, requests, validation, generation units, generated-pack cache, workbench cache, reset handling, and coverage-aware finalization.
- **Runtime composition cleanup:** Large runtime wiring has been moved into focused composition modules for the runtime shell, Context, Loredeck editor, and Loredeck workflows, reducing the central runtime control surface while preserving the current UI behavior.
- **Alpha gate expansion:** The alpha release gate now covers the new bundled pack families, Creator generation modules, runtime composition modules, Story Maker timeout/cancellation paths, and search-focus preservation contracts.

### Minor Changes

- **Workbench search focus:** Search boxes in the Context Workbench, Lore Workbench, Loredeck Workbench, and Deck Maker project shelf now preserve focus and caret position across rerenders.
- **Lorecards polish:** Lorecards workspace sorting now uses Theme Pack-connected button tokens instead of hardcoded accent styling.
- **Cover asset metadata:** Early One Piece packs and Star Trek TNG/Voyager packs have deck-local cover assets wired through bundled manifests and the bundled index/defaults.
- **Mobile documentation renders:** Mobile documentation screenshots were refreshed for Basic settings, shell start, Advanced settings, provider setup, Context proposals, and Injection placement.
- **Release documentation alignment:** The README and documentation index now point at alpha 3 as the current release note.
- **Refactor planning:** A new Deep Refactor Target Audit records the current highest-value cleanup targets after the alpha 2 mobile work and the alpha 3 bundled-content expansion.
- **Development-doc cleanup:** Temporary bundled Loredeck authoring plans, review reports, and generated evidence dumps were removed after their content landed in the bundled packs.
- **Metadata cleanup:** A small set of bundled-pack descriptions was normalized to ASCII punctuation.

## Alpha Boundaries

- First run does not auto-open Saga; open it from the extension menu.
- Schemas and storage formats are still changing during pre-alpha.
- Legacy state from older local builds is not guaranteed to migrate. Export anything important before updating.
- Unsupported old imported state schemas are rejected instead of partially migrated.
- Imported Loredeck packages are data-only, but their lore can still affect prompts after users load and use them. Treat packages from unknown sources as untrusted prompt material.
- Direct provider key storage is available for alpha testing, but SillyTavern Connection Profiles or a backend proxy are preferred when possible.
