# Saga 0.1.0-alpha.1 - "Fox One" Release Notes

Saga `0.1.0-alpha.1`, code-named **Fox One**, is a pre-alpha build. Set your expectations to *alpha*.

Last updated: June 13, 2026.

## Compatibility

- Minimum SillyTavern version: `1.12.0`.
- `auto_update: false`; automatic updates are disabled for this alpha.
- Update manually from the repository when moving to a newer alpha build.

## Implemented Features

- **Runtime shelf:** Saga opens from the SillyTavern extension menu and provides the main runtime surface for Session, Loredecks, Context, Lorecards, Continuity, Injection, and Settings.
- **Basic and Advanced modes:** Basic provides the guided first-use path with the Session Start Checklist and Basic Walkthrough. Advanced exposes the full operator tools, diagnostics, Creator workflows, Pack Health, provider settings, Continuity, Injection controls, and State Safety.
- **Loredeck Library:** Users can browse Bundled, Generated, and Custom Loredecks; organize Library folders; inspect deck details; manage the Active Stack; import packages; export decks; and run Pack Health.
- **Loredeck Creator:** Advanced users can create Generated Loredecks through a staged workflow covering scope, outline, Context planning, title batches, timeline/tag planning, Lorecard drafting, review, Pack Health, and finalization.
- **Lorecard system:** Saga supports suggested lore, Scan Story Lore, auto-relevance suggestions, Pending Review, approved Lorecards, pin/mute controls, accepted-lore editing, and Lore Timeline recovery.
- **Context system:** Saga tracks story position for loaded Loredecks and supports Context Workbench tools for selecting windows, anchors, before/after positions, timeline points, and phrase-based resolution.
- **Continuity tools:** Saga can scan recent or selected chat ranges for story-state updates and feed structured continuity into runtime decisions.
- **Injection system:** Saga can assemble prompt-ready lore from the Active Stack, current Context, accepted Lorecards, relevance scoring, continuity state, and compression settings.
- **Pack Health Center:** Users can scan Loredecks for structural issues, review grouped findings, repair supported problems, accept intentional issues as-is, and verify fixes.
- **Loredeck packages:** Saga imports and exports `.saga-loredeck.zip` packages with manifests, entry files, registries, Library placement metadata, and passive cover assets.
- **Theme and icon customization:** Users can import and manage custom Theme Packs and Icon Sets while bundled appearance packs remain available.
- **Storage and State Safety:** Saga stores large custom payloads in SillyTavern `/user/files` where available, keeps `settings.json` compact, verifies storage, settles queued writes, and provides scoped cleanup tools.
- **Provider setup:** Saga supports the current SillyTavern model, SillyTavern Connection Profiles, and direct OpenAI-compatible endpoints for Utility and Reasoning provider roles.
- **Release-facing documentation:** The README, Basic Workflow, Advanced Workflow, Operator's Manual, Storage And State Safety guide, Loredeck authoring docs, and Loredeck zip package guide are available for testers.

## Alpha Boundaries

- First run does not auto-open Saga; open it from the extension menu.
- Schemas and storage formats are still changing during pre-alpha.
- Legacy state from older local builds is not guaranteed to migrate. Export anything important before updating.
- Unsupported old imported state schemas are rejected instead of partially migrated.
- Imported Loredeck packages are data-only, but their lore can still affect prompts after users load and use them. Treat packages from unknown sources as untrusted prompt material.
- Direct provider key storage is available for alpha testing, but SillyTavern Connection Profiles or a backend proxy are preferred when possible.
