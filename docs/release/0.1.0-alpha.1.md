# Saga 0.1.0-alpha.1 Release Notes

Saga `0.1.0-alpha.1` is a pre-alpha integration-hardening build. It is meant for controlled tester installs, not broad distribution.

## Compatibility

- Minimum SillyTavern version: `1.12.0`.
- `auto_update: false`; update manually from the repository when moving to a newer alpha build.
- Run `node tools/scripts/run-alpha-gate.mjs` before tagging or packaging this build.

## State And Updates

Saga schemas are still changing. This alpha does not promise legacy-state compatibility for old local builds.

- Export Saga state before updating.
- State Safety creates backups before imports, restores, lifecycle cleanup, creator finalization, review bulk actions, and destructive resets.
- State Safety also owns Saga storage migration, `/user/files` verification, queued-write settling, and missing-record cleanup for externalized payloads.
- Unsupported old imported state schemas are rejected with a clear error instead of partial migration.
- If a tester is carrying stale pre-alpha state, prefer exporting anything still useful, resetting Saga-only state, and rebuilding from current Loredecks.

## Alpha Notes

- First run does not auto-open the runtime; open Saga from the extension menu.
- Basic Workflow starts from the Session tab Start Checklist and Basic Walkthrough.
- Advanced mode owns destructive slash bulk actions, full Injection diagnostics, Continuity, Creator workflows, and Pack Health tools.
- Prompt injection clears on chat switches, interrupted or failed generation, disabled injection, and extension disable.
- Imported and generated Saga-owned content is expected to live in flat `/user/files` JSON payloads with compact settings pointers, not as full payloads inside `settings.json`.
- Direct provider key storage is discouraged; use SillyTavern connection profiles or a backend proxy where possible. Diagnostic exports redact provider keys, bearer tokens, endpoint secrets, and provider headers.
