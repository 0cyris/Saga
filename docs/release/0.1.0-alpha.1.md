# Saga 0.1.0-alpha.1 Release Notes

Saga `0.1.0-alpha.1` is a pre-alpha integration-hardening build for controlled tester installs. It is not intended for broad distribution yet.

Last updated: June 13, 2026.

## Compatibility

- Minimum SillyTavern version: `1.12.0`.
- `auto_update: false`; update manually from the repository when moving to a newer alpha build.
- Run `node tools/scripts/run-alpha-gate.mjs` before tagging or packaging this build.

## Current Alpha Scope

This alpha now covers the full runtime shelf loop: Basic and Advanced workflows, Loredeck Library, Active Stack, Context, Lorecards, Lore Timeline, Injection, Continuity, Loredeck Creator, Pack Health Center, package import/export, custom Theme Packs, custom Icon Sets, and external Saga storage.

The recommended tester path is:

1. Install or update manually from the repository.
2. Open Saga from the extension menu; first run does not auto-open the runtime.
3. Start in **Basic**, use the **Session** Start Checklist, and get one chat working.
4. Switch to **Advanced** when you need Creator authoring, Pack Health repairs, Continuity, provider configuration, full Injection diagnostics, or State Safety maintenance.

## Storage And State Safety

Saga schemas are still changing. This alpha does not promise legacy-state compatibility for old local builds.

- Imported/generated Loredecks, Creator projects, custom Theme Packs, custom Icon Sets, and passive assets are expected to live in flat SillyTavern `/user/files` payloads with compact settings pointers.
- `settings.json` should keep preferences, provider selections, storage pointers, lightweight diagnostics, and direct-key metadata when used. Full Loredeck entries, Creator stage artifacts, large Theme Pack libraries, and base64 cover/icon blobs should not live there.
- State Safety creates backups before imports, restores, lifecycle cleanup, Creator finalization, review bulk actions, and destructive resets.
- State Safety owns `/user/files` verification, queued-write settling, stale-write detection, missing-record cleanup, and storage diagnostics.
- Settings has a bottom **Danger Zone** split into Active Chat and Global cleanup. Global cleanup can remove custom Theme/Icon packs, remove custom/imported/generated Loredecks, reset all settings and stored Saga API keys, or run typed-confirmation **Total Saga Cleanup**.
- Saga no longer supports migrating older settings-backed payloads. Unsupported old imported state schemas are rejected with a clear error instead of partial migration.
- If a tester is carrying stale pre-alpha state, export anything still useful, run the current Saga cleanup path, and rebuild from current Loredecks.

## Security And Provider Access

- Saga is browser-side and does not require a server plugin for its own storage model.
- Loredeck packages are data-only zip archives. Package import rejects unsafe paths and active file types such as scripts, HTML, SVG, executables, shell scripts, and WebAssembly.
- Imported Loredecks can still affect prompt content after users load and use them, so packages from unknown sources should be treated as untrusted prompt material.
- Model-backed actions use separate **Utility Provider** and **Reasoning Provider** roles.
- Provider routes are explicit: current SillyTavern model, SillyTavern Connection Profile, or direct OpenAI-compatible endpoint.
- Direct OpenAI-compatible endpoint calls use browser `fetch` with `credentials: 'omit'` and a bearer API key header.
- Direct provider key storage is discouraged. Use SillyTavern Connection Profiles or a backend proxy where possible. When direct keys are used, Saga stores them with browser WebCrypto AES-GCM when available; if WebCrypto is unavailable, Saga warns and uses fallback obfuscation instead of encryption. Decrypted keys live in browser memory during provider calls.
- Diagnostic exports redact provider keys, bearer tokens, endpoint secrets, and provider headers.

## Loredeck Library And Packages

- Public Loredeck import/export is `.saga-loredeck.zip`.
- Package import installs package decks as editable Custom Loredecks and preserves package metadata, folder placement, entry files, registries, and passive cover assets where valid.
- Package export creates a data-only zip containing `saga-package.json`, `loredecks/index.json`, deck folders, manifests, entry files, registries, and passive assets.
- Bundled Loredecks remain exportable as zip packages, but package import cannot overwrite bundled content.
- Uploaded Loredeck cover assets are preserved through external payload hydration before save.
- Duplicate and collision review reports exact imported and skipped pack IDs instead of inferring status from selected rows.
- The Library now keeps cover-strip layout stable, caps visible folder cover previews, and avoids random cover tilting.

## Loredeck Creator

- Creator is a staged, review-first workflow: Scope Brief, Story Outline and Context Plan, Title Pass, Timeline and Tag Planning, Lorecard Drafting, Review, Pack Health, and finalization.
- Creator Lorecard drafting distinguishes provider failures from Saga-side schema rejections, preserves valid drafts from mixed batches, surfaces compact preflight/rejection details, and can use **Auto split failed batches** to retry affected titles in smaller batches.
- Creator drafting keeps small default micro-batches, exposes **Draft Lorecards** and **Auto-Draft All**, and confirms the exact remaining Lorecard count before repeated generation.
- Long-running Creator drafting now shows visible busy-button progress and recomputes remaining work from fresh eligibility state.
- Creator title selection refreshes row, count, and button state without rebuilding the whole workbench or losing generation-locked button state.
- Creator project cards and job status rows use shared job-view components, and in-progress project activation hydrates external project payloads after reload.
- Generated Loredeck progress now hydrates external payload-backed generated packs for Creator coverage, readiness, planning, and review surfaces after reload.
- Creator Coverage can block finalization when coverage is missing or intentionally light until the user expands coverage or explicitly acknowledges finalizing anyway.
- Creator warning cleanup removed moralizing/advisory warning language from prompt contracts and visible Creator state while preserving technical parser, rejection, and recovery diagnostics.

## Pack Health Center

- Pack Health Center is now the release gate for Loredeck reliability across manifests, embedded metadata, entry structure, tag registries, timeline references, Context gates, stats, and package assets.
- **Refresh Scan** and Health Center actions use shared busy-button/action-row helpers.
- Grouped issue repair is centered on **Attempt Fixing**, **Accept As-Is**, and **Verify Fixed** flows.
- Storage-backed repair sessions persist repair state outside `settings.json` and are included in State Safety cleanup scope.
- Stage 2 repair wiring hydrates external payload-backed Loredecks before planning or applying repairs, waits for payload/library writes, and refreshes visible Health Center state after repair.
- The removed assistant repair drafting path no longer appears in Pack Health UI; model-backed repairs route through the Pack Health repair runner and repair contracts.

## Workflow And Runtime Notes

- Basic Workflow starts from the Session tab Start Checklist and Basic Walkthrough.
- Advanced mode owns full Injection diagnostics, Continuity, Creator workflows, Pack Health tools, package import/export, provider configuration, and the advanced State Safety maintenance surface.
- Basic Loredecks keeps **Import Deck** and Library access while hiding advanced Creator project and repair controls.
- Basic Start Checklist actions can launch focused external mini-tours for the relevant workflow step.
- Prompt injection clears on chat switches, interrupted or failed generation, disabled injection, and extension disable.
- Lore Timeline is the audit and recovery surface for accepted Lorecard changes, deleted versions, restored entries, and continuity-related lore events.
