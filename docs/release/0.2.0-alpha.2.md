# Saga 0.2.0-alpha.2 - "The Big Mobile Update" Release Notes

Saga `0.2.0-alpha.2`, code-named **The Big Mobile Update**, is an alpha build. Set your expectations to *alpha*.

Last updated: June 16, 2026.

## Compatibility

- Minimum SillyTavern version: `1.12.0`.
- `auto_update: false`; automatic updates are disabled for this alpha.
- Update manually from the repository when moving to a newer alpha build.
- This build follows `0.1.0-alpha.1` (**Fox One**). Export anything important before updating from older local builds.

## Implemented Features

### Major Changes

- **Mobile UI:** Saga now has a phone-width shell with bottom navigation, route pages, mobile subviews, detail sheets, touch-sized controls, and long-press editing paths. Basic and Advanced workflows are both usable on phone-width screens.
- **Story Maker:** Users can create saved opener sessions from the Session tab, build a Context Packet, generate an Opener Brief, draft variants, revise the selected opener, and copy lore-aware opening prose.
- **Mobile Loredeck workflows:** The Library, Active Stack, Pack Health, Loredeck detail sheets, Context, Lorecards, Settings, and Advanced-only Continuity/Injection routes now have mobile operating paths instead of relying on desktop panes squeezed into a phone viewport.
- **Mobile Lorecards:** Phone-width Lorecards use `Lore`, `Generate`, and Advanced `Automate` sub-tabs, object rows, long-press editing, touch-friendly relevance controls, Pending Review, and accepted-lore review.
- **Expanded documentation:** The release-facing docs now include separate desktop and mobile operator manuals, separate Story Maker and Deck Maker guides for desktop and mobile, updated Basic/Advanced workflows, and a larger screenshot-backed documentation set.

### Minor Changes

- **Deck Maker rebrand:** The old Loredeck Creator surface is now named **Deck Maker**. This is a product-language cleanup, not a newly added feature.
- **Deck Maker workflow polish:** Deck Maker has stronger staged progress, current-task guidance, draft review, retry recovery, readiness checks, Pack Health handoff, reset-to-stage support, and finalization handling.
- **Lore Automation levels:** Advanced users can choose Lore Automation authority levels and inspect/review automation behavior while keeping accepted lore reviewable and reversible.
- **Elevate and Mute controls:** Accepted Lorecards now use clearer relevance, **Elevate**, and **Mute** language in the public UI and docs.
- **Context workflow coverage:** The Context Workbench and Phrase Resolver are covered more directly in the Basic and Advanced workflows, including `Start Here`, `Use Window`, `Use Anchor`, `After`, `Before`, `Timeline`, and `Phrase Resolver`.
- **Pack Health and Library polish:** Library opening, folder workflows, deck details, package import/export, Pack Health review, repairs, accepted issues, and verification paths have been tightened across desktop and mobile.
- **Settings and State Safety:** Provider setup, Theme Packs, Icon Sets, State Safety, stale-write checks, storage diagnostics, and Danger Zone cleanup are documented and routed through the current Settings surfaces.
- **Prompt and injection controls:** Advanced Injection preview, placement, compression, and Context-blocked accepted lore behavior have better coverage and regression checks.

## Alpha Boundaries

- First run does not auto-open Saga; open it from the extension menu.
- Schemas and storage formats are still changing during pre-alpha.
- Legacy state from older local builds is not guaranteed to migrate. Export anything important before updating.
- Unsupported old imported state schemas are rejected instead of partially migrated.
- Imported Loredeck packages are data-only, but their lore can still affect prompts after users load and use them. Treat packages from unknown sources as untrusted prompt material.
- Direct provider key storage is available for alpha testing, but SillyTavern Connection Profiles or a backend proxy are preferred when possible.
